 Level 2 - Walkthrough
========================================================================
 Category   : Reverse Engineering (RE)
 Difficulty : Easy
 Binary     : level2.exe  (PE32+ / Windows x64, ~19 KB)
 Goal       : Recover the secret key that prints
              "[+] Access Granted! You solved it."

 ***  ANSWER: the secret key is  ->  reverse  <-  ***

 The key is NOT stored in plaintext. It is built on the stack as
 obfuscated bytes and checked with a per-character XOR 0x5A. Recovered
 by static disassembly - no debugger, no brute force.
========================================================================

------------------------------------------------------------------------

    -> level2.exe

    file level2.exe

3. LOCATING THE CHECK
------------------------------------------------------------------------
No symbols, so anchor on a string we know. "Enter Secret Key:" lives in
.rdata; the code that references it is main.

Using pefile + capstone, resolve sections then scan .text for a
RIP-relative lea to that string's VA:

    ImageBase 0x140000000
    .text  VA 0x140001000
    .rdata VA 0x140003000   (UI strings)
    .data  VA 0x140005000

The reference lands at 0x140001371, inside the function at 0x140001300.
The comparison loop:

    ; expected key is a byte range [start, end)
    0x140001395  mov  rax, [0x140005220]   ; end pointer
    0x14000139c  mov  r8,  [0x140005218]   ; start pointer
    0x1400013a3  sub  rax, r8              ; rax = expected length
    ...
    0x1400013b0  mov  rdx, [rsp+0x30]      ; rdx = input length
    0x1400013b5  cmp  rdx, rax
    0x1400013b8  jne  fail                 ; length must match first

    loop:                                  ; rax = index
    0x1400013ce  movzx ecx, byte [input + rax]
    0x1400013d2  xor   cl, 0x5a            ; <-- transform input byte
    0x1400013d5  cmp   cl, byte [r8 + rax] ; compare to expected[i]
    0x1400013d9  jne   fail
    0x1400013db  inc   rax
    0x1400013de  cmp   rax, rdx
    0x1400013e1  jb    loop

Whole algorithm in one line:

    (input[i] XOR 0x5A) == expected[i]   for every i

Invert it to forge a valid key:  input[i] = expected[i] XOR 0x5A.
Now we need expected[].


4. RECOVERING THE EXPECTED BYTES
------------------------------------------------------------------------
expected is pointed to by [0x140005218] (start) / [0x140005220] (end).
Those addresses are in .data but PAST the section's raw size - i.e.
uninitialized on disk, filled at runtime by a static initializer (a
global string built during CRT startup). Reading file bytes there gives
nothing.

So find the initializer: scan .text for the instruction that WRITES
0x140005218. It is at 0x140001026; the surrounding code builds the bytes
as stack immediates before copying them into the allocated buffer:

    0x140001000  sub  rsp, 0x28
    0x140001004  mov  ecx, 7                            ; length = 7
    0x140001009  mov  dword ptr [rsp+0x30], 0x3f2c3f28  ; bytes 0-3
    0x140001011  mov  word  ptr [rsp+0x34], 0x2928      ; bytes 4-5
    0x140001018  mov  byte  ptr [rsp+0x36], 0x3f        ; byte 6
    0x14000101d  call 0x140001cb0                       ; allocate
    0x140001026  mov  [0x140005218], rax                ; store start ptr
    0x140001038  mov  [rax], ecx                        ; write the bytes
    ...

Reassemble the immediates little-endian -> 7 expected bytes:

    0x3f2c3f28 -> 28 3f 2c 3f
    0x2928     -> 28 29
    0x3f       -> 3f
    => 28 3f 2c 3f 28 29 3f


5. DECODE
------------------------------------------------------------------------
XOR each expected byte with 0x5A:

    enc = bytes.fromhex('283f2c3f28293f')
    print(bytes(b ^ 0x5A for b in enc).decode())

    expected  ^0x5A  char
    0x28      0x72   r
    0x3f      0x65   e
    0x2c      0x76   v
    0x3f      0x65   e
    0x28      0x72   r
    0x29      0x73   s
    0x3f      0x65   e

    Result:  reverse

ANSWER
------------------------------------------------------------------------
    Secret Key   : reverse
    Obfuscation  : per-byte XOR 0x5A against a stack-built byte array
    Method       : static disassembly (pefile + capstone) + invert XOR


TOOLS USED
------------------------------------------------------------------------
    unzip     - extract the archive (password: crackmes.one)
    file      - identify format/architecture (PE32+ x64)
    Python re - `strings` substitute; confirm there's no plaintext key
    pefile    - parse PE headers, resolve section VAs, read data at any RVA
    capstone  - x86-64 disassembly of the check + initializer
    the binary- dynamic confirmation of the recovered key

Equivalent GUI tooling: Ghidra / IDA / Binary Ninja would decompile the
same loop. For a check this small, a scripted capstone pass beats opening
a full disassembler.


TAKEAWAYS / METHODOLOGY
------------------------------------------------------------------------
  1. No plaintext secret != dead end. Absence of std::string/strcmp plus
     presence of strlen/memcpy fingerprints a hand-rolled byte comparison.
  2. Anchor on known strings. With no symbols, find main by locating the
     lea that references a UI string you already see printed.
  3. Read the transform, then invert it. Once you see
     `xor cl,0x5A; cmp cl,[expected]`, you don't need the key in the file:
     key[i] = expected[i] XOR 0x5A.
  4. Runtime-initialized globals live in code, not data. When a .data
     pointer sits past the raw section size, go find the initializer that
     fills it - the constant bytes are usually mov-immediates on the stack.
